# artemis-financial-security-analysis
Cybersecurity-focused project demonstrating secure software practices, vulnerability assessment, and risk mitigation for a financial services application.

Project Overview:
This artifact is my Artemis Financial Practices for Secure Software Report from Project Two. The purpose of this project was to analyze a financial services company’s software and improve its security by identifying vulnerabilities and applying secure coding practices.

Client Summary:
Artemis Financial is a company that provides financial services and handles sensitive customer data, including personal and financial information. Because of this, security is a major concern. The main issue the company wanted addressed was protecting its application from potential cyber threats, such as data breaches or unauthorized access. The goal was to ensure secure communication, protect stored data, and strengthen the overall reliability of the system.

Identifying Vulnerabilities and Importance of Secure Coding:
One thing I did well during the vulnerability assessment was carefully reviewing dependencies and identifying outdated or insecure components in the project. I also focused on understanding how vulnerabilities could impact real-world users, especially in a financial environment.
Coding securely is important because even small weaknesses can be exploited by attackers. Software security adds value to a company by protecting customer trust, preventing financial loss, and helping the organization meet legal and compliance requirements.

Challenges and Learning Experience:
One of the most challenging parts of the vulnerability assessment was understanding how different security tools work and interpreting their results. For example, reviewing dependency reports and deciding which vulnerabilities were critical required attention to detail. However, this was also one of the most helpful parts because it gave me hands-on experience with real security tools and how they are used in the industry.

Increasing Security Layers:
To improve security, I implemented multiple layers of protection, including encryption, secure communication using HTTPS, and stronger dependency management. I also reviewed code for potential weaknesses and applied best practices to reduce risk.
In the future, I would use tools like automated vulnerability scanners, static code analysis tools, and frameworks that follow secure coding standards. These tools help prioritize risks and guide decisions on how to mitigate them effectively.

Ensuring Functionality and Security:
To make sure the application remained functional and secure, I tested the system after making changes to confirm that everything still worked correctly. After refactoring the code, I reran security scans to check for any new vulnerabilities that may have been introduced. This helped ensure that improvements did not accidentally create new issues.

Tools and Resources Used:
Throughout this project, I used several tools and practices that will be helpful in the future, including:
-OWASP Dependency-Check for identifying vulnerable libraries
-Eclipse IDE for development and testing
-Maven for project and dependency management
-Secure coding practices such as encryption and input validation
These tools are important for maintaining secure and reliable software.
